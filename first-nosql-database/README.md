# AWS SimuLearn: First NoSQL Database

## Overview
This guided AWS lab focused on the fundamentals of NoSQL databases using Amazon DynamoDB. The exercise introduced how to create a DynamoDB table, work with records using a flexible schema, and query stored data.

## Objective
To understand the basic structure and use of a NoSQL database in AWS by creating and updating a DynamoDB table and validating data changes.

## Lab Context
The practice lab focused on:
- creating a NoSQL database as an Amazon DynamoDB table
- adding records with dynamic schema to the DynamoDB table
- querying the DynamoDB table

## Tasks Performed
- Reviewed the DynamoDB lab scenario and data model
- Worked with the **UserVideoHistory** table
- Created a new item in the table with a unique **userId** value
- Added a new **Number** attribute called **rating** to the created record
- Validated that the new attribute existed correctly in the table item

## AWS Services and Concepts Involved
- Amazon DynamoDB
- NoSQL database concepts
- Table creation
- Items and attributes
- Primary key structure
- Dynamic schema
- Querying records

## What I Learned
- How DynamoDB stores data in tables, items, and attributes
- The difference between structured relational thinking and flexible NoSQL schema design
- How primary keys help identify records in DynamoDB
- How new attributes can be added dynamically at item level
- How DynamoDB supports scalable and flexible data storage patterns

## Business Relevance
NoSQL databases are useful for applications that need flexible schemas, fast performance, and scalability. This lab reflects a common real-world scenario where user activity data can evolve over time without requiring a rigid relational structure.

- Basic DynamoDB understanding
- NoSQL data modeling awareness
- Table and item creation
