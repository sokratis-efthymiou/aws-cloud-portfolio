# AWS SimuLearn: Computing Solutions

## Overview
This guided AWS lab focused on Amazon EC2 instance types, connection methods, instance metadata, and basic instance resizing. The exercise introduced how to explore EC2 options, connect to an instance through different AWS tools, and validate a change in instance size.

## Objective
To understand core EC2 operational concepts by reviewing instance types, accessing an EC2 instance, and configuring an instance type change.

## Lab Context
The practice lab focused on:
- exploring Amazon EC2 instance types
- filtering EC2 instances based on their attributes
- connecting to an EC2 instance by using EC2 Instance Connect
- viewing EC2 instance metadata
- starting and stopping an EC2 instance by using the Amazon EC2 console

## Tasks Performed
- Reviewed the lab architecture and EC2 interaction flow
- Explored EC2 instance access methods through:
  - EC2 Instance Connect
  - AWS Systems Manager
  - Terminal access flow
- Reviewed instance metadata access
- Worked with an EC2 instance initially shown as **t3.micro**
- Configured the Amazon EC2 instance type change to **m4.large**
- Validated that the target instance type was configured correctly

## AWS Services and Concepts Involved
- Amazon EC2
- EC2 instance types
- EC2 Instance Connect
- AWS Systems Manager
- Instance metadata
- Instance resizing
- Basic operational management of compute resources

## What I Learned
- How EC2 instance types differ based on workload characteristics
- How to filter and evaluate EC2 instance options
- How AWS provides multiple methods for connecting to compute resources
- How instance metadata can be used to inspect configuration details
- How resizing an instance affects the compute configuration
- Why instance type selection matters for performance and operational needs

## Business Relevance
Compute sizing is a core decision in cloud operations. Organizations must choose instance types that match workload requirements while balancing performance, manageability, and cost. This lab reflects a practical cloud operations scenario where an instance is reviewed and reconfigured to better fit a target use case.
