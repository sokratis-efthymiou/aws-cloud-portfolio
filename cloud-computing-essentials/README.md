# AWS SimuLearn: Cloud Computing Essentials

## Overview
This guided AWS lab focused on Amazon S3 static website hosting and basic access configuration through a bucket policy. The exercise introduced how cloud storage can be used to host simple web content and how object-level access affects website behavior.

## Objective
To understand how to configure an Amazon S3 bucket for static website hosting and manage access to hosted content by working with bucket policies and object names.

## Lab Context
The practice lab focused on:
- enabling static website hosting on an Amazon S3 bucket
- reviewing a bucket policy to secure the hosting bucket
- understanding how web content is retrieved from an S3-hosted website
- managing hosted objects in the website bucket

## Tasks Performed
- Reviewed the static website hosting architecture using Amazon S3
- Examined how city residents accessed the website through a web portal
- Reviewed the role of the S3 bucket policy in allowing access to hosted objects
- Worked with the static website bucket and hosted HTML files
- Renamed `index.html` to `waves.html`
- Validated that the renamed object existed correctly in the S3 bucket

## AWS Services and Concepts Involved
- Amazon S3
- Static website hosting
- Bucket policy
- Object storage
- Access configuration
- Basic web hosting concepts

## What I Learned
- How Amazon S3 can be used to host static website content
- How object names affect the availability of hosted web pages
- How bucket policies help control access to S3-hosted content
- How static website hosting works differently from traditional compute-based web hosting
- Why secure and correctly configured storage access is important for public-facing content

## Business Relevance
Static website hosting on Amazon S3 is a common low-cost cloud pattern for lightweight websites, landing pages, and public information portals. This lab reflects a practical scenario where storage configuration, object naming, and access policy directly affect how users can retrieve hosted content.
