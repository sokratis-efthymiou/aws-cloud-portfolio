# AWS SimuLearn: Cloud Economics

## Overview
This guided AWS lab focused on cloud cost awareness and infrastructure rightsizing. The exercise introduced the relationship between instance sizing and overall cloud spending by using an AWS pricing estimation scenario for Amazon EC2 resources.

## Objective
To understand how infrastructure choices affect cloud costs and to apply a simple rightsizing action in a cloud pricing estimate.

## Lab Context
The practice lab focused on:
- creating logical pricing groups
- creating an estimate for Amazon EC2 usage
- reviewing the impact of changing instance size on the total estimate

## Tasks Performed
- Reviewed the initial lab architecture and pricing scenario
- Worked with an Auto Scaling group containing EC2 web server instances
- Updated the EC2 instance type in the estimate from **t3.medium** to **t2.micro**
- Generated a new pricing estimate link after the rightsizing change
- Validated that the estimate reflected the updated configuration

## AWS Services and Tools Involved
- Amazon EC2
- Auto Scaling concepts
- AWS Pricing Calculator

## What I Learned
- How EC2 instance type selection affects estimated cloud cost
- Why rightsizing is important in cloud environments
- How cloud pricing estimates support cost-conscious decision-making
- The importance of reviewing infrastructure sizing before deployment

## Business Relevance
In enterprise cloud environments, choosing the right instance type is important for balancing performance and cost. This lab introduced a basic FinOps mindset by showing how cloud costs can be reduced through better sizing decisions and more appropriate infrastructure estimates.

- Basic AWS cost awareness

- Use of AWS pricing estimation tools
- Cost optimization thinking
- Awareness of scalable infrastructure pricing considerations
